All endpoints require an API key sent as a Bearer token in the `Authorization` header.

```mdx
Authorization: Bearer <your_api_key>
```

## Quick Start

### Prerequisites

- Node.js / Python (or any backend language)
- Basic knowledge of HTTP requests
- An API key from [Cortex](https://usecortex.ai/) (`Authorization: Bearer <your_api_key>`)
- A user conversation with an LLM assistant (or use our sample content)

### Step 1: Create a tenant

Create a tenant space for your data:

```mdx
POST /tenants/create
```

```mdx
curl -X POST 'https://api.usecortex.ai/tenants/create' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "Content-Type: application/json" \
  -d '{"tenant_id": "your_tenant_id"}'
```

### **Step 2: Upload memories**

You can upload messages, emails, PDFs, DOCXs, or use the memories API for text/markdown.

#### Option A: Upload a knowledge memory

```mdx
POST /ingestion/upload-knowledge
```

- `memory`: your document
- `tenant_id`: your unique tenant ID

```mdx
curl -X POST 'https://api.usecortex.ai/ingestion/upload-knowledge' \
  -H "Authorization: Bearer <your_api_key>" \
  -F "file=@/path/to/your/document.pdf" \
  -F "tenant_id=your_tenant_id"
```

#### Option B: Upload user memories

Use [Add Memory](/api-reference/endpoint/add-memories) for uploading user conversations, user-LLM pairs, texts, or markdowns

### **(Optional): Verify Processing**

Check if the memories are fully indexed:

```mdx
POST /ingestion/verify-processing?file_id=YOUR_FILE_ID&tenant_id=your_tenant_id
```

**Request Body:**

```json
{
  "question": "What did the team say about pricing?",
  "session_id": "user-session-123",
  "tenant_id": "your_tenant_id",
  "context_list": ["optional_specific_file_id"],
  "tenant_metadata": {"source_title": "contract.pdf"}
}
```

**Example (cURL):**

```mdx
curl -X POST 'https://api.usecortex.ai/search/qna' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "Summarize the contract terms",
    "session_id": "user-123-session",
    "tenant_id": "your_tenant_id",
    "tenant_metadata": {"source_title": "contract.pdf"},
    "document_metadata": {"source_title": "contract.pdf"}
  }'
```
